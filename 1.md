## 1. 项目概述

### 1.1 项目背景

本项目旨在构建一个基于多Agent共识的分布式邮件分类系统，通过集成Paxos共识算法、机器学习分类模型和消息队列技术，实现自动化的邮件分类决策。

### 1.2 核心功能

| 功能模块 | 描述 |
|----------|------|
| 邮件管理 | 邮件的增删改查、分页搜索 |
| 分类中心 | 用户输入邮件→多Agent分类→Paxos共识→结果展示 |
| Agent监控 | 实时监控各Agent状态和统计数据 |
| Paxos日志 | 记录并展示共识过程 |
| 数据统计 | 分类分布、趋势图表 |
| 系统设置 | 配置管理（分类类别、Paxos参数等） |

### 1.3 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML/CSS/JavaScript | 6个功能页面，数据可视化 |
| 后端 | Python + Flask | REST API服务 |
| 数据库 | MySQL | 数据持久化 |
| 消息队列 | RabbitMQ | Agent间异步通信 |
| 共识算法 | Paxos | 分布式决策一致性 |
| 容器化 | Docker | 服务打包 |
| 编排 | Kubernetes | 容器管理 |
| 服务网格 | Istio | 流量管理 |

### 3.2 数据库初始化脚本（init.sql）

```sql
CREATE TABLE emails (
    id INT PRIMARY KEY AUTO_INCREMENT,
    sender VARCHAR(255) NOT NULL,
    subject VARCHAR(500),
    content TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE classifications (
    id INT PRIMARY KEY AUTO_INCREMENT,
    email_id INT NOT NULL,
    agent_name VARCHAR(50),
    method VARCHAR(50),
    category VARCHAR(50),
    confidence FLOAT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (email_id) REFERENCES emails(id)
);

CREATE TABLE paxos_logs (
    id INT PRIMARY KEY AUTO_INCREMENT,
    email_id INT NOT NULL,
    proposal_id INT,
    phase VARCHAR(20),
    proposer VARCHAR(50),
    value VARCHAR(100),
    result VARCHAR(20),
    acceptor_votes JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (email_id) REFERENCES emails(id)
);

CREATE TABLE final_results (
    id INT PRIMARY KEY AUTO_INCREMENT,
    email_id INT NOT NULL UNIQUE,
    category VARCHAR(50),
    method VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (email_id) REFERENCES emails(id)
);

CREATE TABLE system_config (
    id INT PRIMARY KEY AUTO_INCREMENT,
    config_key VARCHAR(100) UNIQUE,
    config_value TEXT,
    updated DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 4. 核心模块设计

### 4.1 Paxos共识算法模块

#### 4.1.1 消息定义（paxos/message.py）

| 消息类型 | 说明 | 数据结构 |
|----------|------|----------|
| PREPARE | Proposer发起准备请求 | {type, proposal_id} |
| PROMISE | Acceptor承诺 | {type, proposal_id, accepted_value} |
| ACCEPT | Proposer发起接受请求 | {type, proposal_id, value} |
| ACCEPTED | Acceptor接受 | {type, proposal_id, value} |
| REJECT | Acceptor拒绝 | {type, proposal_id, reason} |
| LEARN | Learner学习 | {type, value} |

#### 4.1.2 Proposer角色（paxos/proposer.py）

| 方法 | 功能 |
|------|------|
| __init__ | 初始化，生成唯一ID |
| propose | 发起提案 |
| prepare | 发送Prepare请求 |
| handle_promise | 处理Promise回复 |
| accept | 发送Accept请求 |
| handle_accepted | 处理Accepted回复 |
| has_majority | 判断是否获得多数派支持 |

#### 4.1.3 Acceptor角色（paxos/acceptor.py）

| 方法 | 功能 |
|------|------|
| __init__ | 初始化 |
| handle_prepare | 处理Prepare请求 |
| handle_accept | 处理Accept请求 |
| promise | 返回Promise |
| accept | 返回Accepted |
| reject | 返回Reject |

#### 4.1.4 Coordinator协调器（paxos/coordinator.py）

| 方法 | 功能 |
|------|------|
| __init__ | 初始化Proposer和Acceptor |
| run_consensus | 执行完整共识流程 |
| wait_for_response | 等待响应（带超时） |
| get_result | 获取共识结果 |
